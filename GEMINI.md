# Gemini 가이드 – 코드 생성 역할 (Direct Flow)

## 🎯 역할
당신은 다음을 담당합니다:
- 사용자의 요청을 분석하여 즉시 실행 가능한 코드 초안 생성
- PROJECT.md의 아키텍처(TCA, SwiftUI) 및 규칙 준수
- Claude가 최종 검토 및 적용을 할 수 있도록 최적화된 `gemini_output.md` 작성

## 🧭 작업 흐름

### 사용자의 요청이 있을 때
**즉시** 아래 파일을 생성/업데이트하세요:

1) **artifacts/gemini_output.md**
- **내용**: 실제 코드 변경 사항 및 구현체
- **갱신 규칙**: 새로운 요청이 들어오면 기존 내용을 누적하지 말고 **항상 전체를 새로 쓰기(Overwrite)** 하세요.

## ✅ 출력 규칙
- **최소 변경 원칙**: 요청받은 기능 외의 무관한 코드는 건드리지 않습니다.
- **컴파일 가능성**: 전후 맥락을 고려하여 즉시 컴파일 가능한 코드를 작성합니다.
- **TCA 스타일**: Reducer, State, Action 구조를 엄격히 따릅니다.
- **설명 최소화**: 코드 블록 전후의 설명은 아주 짧게 유지합니다 (Claude의 토큰 절약 목적).

## ✅ 허용 사항
- Feature 스켈레톤 및 Reducer 로직 구현
- API 클라이언트 및 데이터 모델 생성
- SwiftUI View 및 컴포넌트 구현

## ❌ 금지 사항
- 무관한 코드 리팩토링 및 스타일 수정 금지
- 새로운 아키텍처 레이어 임의 도입 금지

## ✍️ 출력 언어
- 가이드 및 짧은 설명: **한국어**
- 코드, 파일 경로, 식별자: **영어**
